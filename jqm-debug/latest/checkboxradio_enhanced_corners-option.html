Checkboxradip : "Providing pre-rendered markup"

1. data-enhanced="false" (default) :

=> Classes which is added after runtime: "ui-btn ui-corner-all ui-btn-inherit ui-btn-icon-left ui-checkbox-off"

2. data-enhanced="false" and data-corners="false" :

=> Classes which is added after runtime: the same with the Case 1 above.
-> The checkboxradio widget doesn't the data-corners option yet. So, data-corners="false" isn't applied to the widget.

3. data-enhanced="false" and add "ui-btn ui-btn-inherit ui-btn-icon-left ui-checkbox-off" class manually with removing "ui-corner-all" class :

=> Classes which is added after runtime: the same with the Case 1 above.
-> The "ui-corner-all" class doesn't exist in html tags, nevertheless, you can see that class is added after runtime.

4. data-enhanced="true" (and set "ui-btn ui-btn-inherit ui-btn-icon-left ui-checkbox-off" classes ):

=> Classes which is added after runtime: the classes which is already specified in DOM. (i.e., Not exist ui-corner-all.)
-> Because the checkboxradio widget is set the attribute data-enhanced="true", that widget is skipped these DOM manipulations during instantiation and is applied the classes which is already specified in DOM. So, the "ui-corner-all" is not added after runtime.

5. data-enhanced="true" and data-corners="false" (API):

=> Classes which is added after runtime: the same with the Case 4 above. (i.e., Not exist ui-corner-all.)
-> As you can see from the result, the description for "Providing pre-rendered markup" of Checkboxradio in jQM 1.4 API is guided that user specifies the attribute data-corners="false" to explicit, I think.
